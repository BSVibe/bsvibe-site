## Overview

The BSNexus Kanban board is a **WebSocket-based real-time board** that lets you watch Executors process tasks as they happen. Task cards move between columns automatically without refreshing, and progress and logs update in real time.

![BSNexus Dashboard Kanban Board](/images/screenshots/bsnexus-dashboard.png)

## Columns

The Kanban board consists of 4 columns corresponding to the Task State Machine states:

| Column | State | Description | Card Color |
|--------|-------|-------------|------------|
| **Backlog** | PENDING / ASSIGNED | Tasks waiting or assigned to an Executor | Gray / Blue |
| **In Progress** | IN_PROGRESS | Tasks currently being worked on | Yellow |
| **Review** | IN_REVIEW | Tasks awaiting review after completion | Purple |
| **Done** | COMPLETED | Tasks that passed review | Green |

:::note
FAILED tasks appear in the Backlog column with a red badge. They either re-enter the queue via the automatic retry policy or require manual intervention.
:::

## Task Card Details

Each task card displays the following information:

### Basic Info
- **Task title**: A concise name for the work (e.g., "Implement user auth API")
- **Task description**: Detailed requirements generated by the Architect
- **Task ID**: Unique identifier (e.g., `TASK-001`)

### Execution Info
- **Assigned Executor**: The Executor currently handling the task
- **Progress**: A percentage bar showing work progress (0-100%)
- **Elapsed time**: Time since the task entered IN_PROGRESS

### Relationship Info
- **Dependency badge**: Number of dependencies and their completion status
- **Blocking badge**: Number of tasks waiting on this one to complete
- **Tags**: Classification tags assigned by the Architect (e.g., `frontend`, `api`, `database`)

### Status Info
- **Retry count**: Number of retries after FAILED (when applicable)
- **Error message**: Summary of the most recent failure cause (for FAILED tasks)

## Card Actions

Click a task card to open the detail panel, where you can:

| Action | Description | Available States |
|--------|-------------|------------------|
| **View details** | See full requirements, execution logs, and outputs | All states |
| **View logs** | Stream the Executor's work logs in real time | IN_PROGRESS |
| **View output** | See generated code, file list, and diffs | IN_REVIEW, COMPLETED |
| **Manual retry** | Reset a FAILED task to PENDING | FAILED |
| **Edit task** | Modify the task title, description, or requirements | PENDING, FAILED |
| **Edit dependencies** | Add or remove dependencies | PENDING |
| **Delete task** | Remove the task from the project | PENDING |

## Real-time Updates

### WebSocket Auto-refresh

The Kanban board reflects the following events in real time via WebSocket:

- **State transitions**: Task cards move between columns automatically (with animation)
- **Progress updates**: The progress bar on IN_PROGRESS cards updates live
- **Log streaming**: View Executor work logs in real time from the detail panel
- **New task additions**: Tasks created by the Architect appear on the board instantly
- **Error alerts**: Toast notifications and error badges appear when a task fails

### Concurrent Access

Multiple users viewing the same project all see the same real-time state.

## Filtering and Search

Use the filter bar at the top of the board to narrow down tasks:

| Filter | Description | Example |
|--------|-------------|---------|
| **State** | Show only tasks in a specific state | IN_PROGRESS only |
| **Executor** | Show tasks assigned to a specific Executor | Claude Code Executor |
| **Tag** | Show tasks with a specific tag | `frontend` tag |
| **Search** | Keyword search in titles and descriptions | Search for "auth" |

## Manual Controls

When needed, you can directly manipulate the board:

- **Drag and drop**: Drag task cards to another column to manually change their state.
- **Reorder**: Drag cards within the same column to change priority order.
- **Bulk select**: Select multiple tasks and change their state at once.

:::caution
Manually changing task states can conflict with the State Machine's automatic transitions. Moving an IN_PROGRESS task to COMPLETED will abort the Executor's work.
:::
