BSNexus is an **orchestration platform** where AI agents collaborate to build software. Describe your idea in natural language, and the AI Architect designs the project while multiple Executors work on tasks in parallel.

## What BSNexus Does

| Feature | Description |
|---------|-------------|
| **AI Architect** | Transforms your natural language ideas into structured projects |
| **Automatic task breakdown** | Splits projects into independent work units (Tasks) |
| **Dependency analysis** | Determines execution order based on inter-task dependencies |
| **Parallel execution** | Multiple AI agents (Executors) work on independent tasks simultaneously |
| **Real-time Kanban** | Track all task progress on a WebSocket-based board |

## Quick Start

### Step 1: Create an Account and Sign In

1. Go to [nexus.bsvibe.dev](https://nexus.bsvibe.dev) in your browser.
2. Click the **Sign in with BSVibe** button.
   - If you don't have a BSVibe account yet, sign up at [auth.bsvibe.dev](https://auth.bsvibe.dev).
3. After signing in, you'll see the dashboard.

![BSNexus Dashboard — View your project list and status summary at a glance.](/images/screenshots/bsnexus-dashboard.png)

The dashboard displays:
- **My Projects**: List of projects you've created
- **Recent Activity**: Recent task changes
- **Active Executors**: Number of agents currently working
- **Quick Stats**: Completion rate, in-progress tasks, and total projects

### Step 2: Create a Project

1. Click the **+ New Project** button in the top right of the dashboard.
2. Enter a project name (e.g., `todo-app`, `blog-platform`).
3. Optionally add a description.
4. Click **Create** to create the project and navigate to the Architect conversation.

### Step 3: Talk to the Architect

On the Architect page, describe what you want to build in natural language. The more specific you are, the better the generated tasks will be.

![Architect conversation — Design your project through AI-powered dialogue.](/images/screenshots/bsnexus-architect.png)

**Effective prompt examples:**

> "Build a to-do web app. React + TypeScript frontend, FastAPI backend, PostgreSQL database. Use JWT-based authentication."

> "Build a blog platform with a markdown editor, category/tag management, and SEO optimization. Use Next.js."

> "Build a real-time chat app using WebSocket, with channel creation/join and message search. Python + Redis pub/sub architecture."

The Architect goes through the following steps:

1. **Requirement analysis**: Parses your description to extract key features.
2. **Clarifying questions**: Asks follow-up questions if anything is ambiguous (e.g., "Which authentication method would you prefer?").
3. **Project structure generation**: Designs the directory structure, tech stack, and module layout.
4. **Task breakdown**: Splits the project into independent tasks and sets up dependencies.
5. **Executor assignment**: Automatically assigns a suitable Executor to each task.

:::tip[Prompt Tips]
- Specifying the tech stack produces more accurate results.
- List the core features explicitly.
- The more detail you provide when answering the Architect's questions, the better the outcome.
:::

### Step 4: Review and Execute Tasks

Once the Architect generates tasks, a Kanban board is automatically set up on the **Board** tab:

1. Review the generated task list.
2. Modify, add, or remove tasks if needed.
3. Click **Start Execution** to have the Executors begin working.
4. Monitor each task's progress in real time on the Kanban board.

### Step 5: Review Results

When all tasks are complete:
- Review each task's output (code, files, etc.).
- Request additional changes from the Architect if needed.
- Download the project results or push them to a connected repository.
