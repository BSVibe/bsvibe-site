## 모델 관리

대시보드 **Models** 페이지에서 LLM 모델을 관리합니다.

### 모델 추가

| 필드 | 설명 |
|------|------|
| Provider | 모델 제공자 (Anthropic, OpenAI, Ollama 등) |
| Model ID | 요청 시 사용할 모델 식별자 |
| API Key | Provider API 키 (AES-256-GCM 암호화 저장) |
| Max Tokens | 최대 출력 토큰 (선택) |

### Alias 설정

자주 쓰는 모델에 짧은 별명을 부여할 수 있습니다:

| Alias | Target Model |
|-------|-------------|
| `fast` | gpt-4o-mini |
| `smart` | claude-sonnet |

## 라우팅 규칙

대시보드 **Rules** 페이지에서 라우팅 규칙을 관리합니다.

### 규칙 추가

1. **Add Rule** 클릭
2. Pattern 입력 (glob 문법 지원)
3. 대상 모델 선택
4. 우선순위 설정 (높을수록 먼저 평가)

### Auto 라우팅

`model: "auto"`로 요청하면 분류기가 복잡도를 판단합니다:

| 분류 전략 | 설명 |
|----------|------|
| Static | 키워드/패턴 기반 휴리스틱 |
| LLM | 로컬 LLM 기반 분류 |
| ML | 학습된 모델 기반 분류 |

분류기 전략과 복잡도별 모델 매핑은 **Settings** 페이지에서 설정합니다.

## API 키 관리

대시보드 **API Keys** 페이지에서 키를 생성하고 관리합니다.

### 키 속성

| 속성 | 설명 |
|------|------|
| 이름 | 키를 식별하는 라벨 |
| Rate Limit | 분당 최대 요청 수 |
| 만료일 | 키 자동 만료 날짜 (선택) |

## CORS 설정

**Settings** 페이지에서 허용할 도메인을 설정합니다. 프론트엔드에서 직접 호출하는 경우 도메인을 추가해야 합니다.

## API 엔드포인트

| 엔드포인트 | 설명 |
|-----------|------|
| `https://gateway.bsvibe.dev/v1/chat/completions` | OpenAI 호환 채팅 API |
| `https://gateway.bsvibe.dev/v1/models` | 등록된 모델 목록 |
| `https://gateway.bsvibe.dev/health` | 서비스 상태 확인 |
