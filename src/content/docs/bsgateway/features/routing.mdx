## 라우팅 규칙 설정

대시보드의 **Routing** 페이지에서 라우팅 규칙을 정의합니다.

![라우팅 규칙 페이지](/images/screenshots/bsgateway-rules.png)

### Passthrough 모델

**Models** 페이지에서 등록된 모든 모델은 자동으로 passthrough 대상이 됩니다.

![모델 관리 페이지](/images/screenshots/bsgateway-models.png)

등록 예시:

| 모델 ID | 프로바이더 |
|---------|----------|
| claude-sonnet | Anthropic |
| gpt-4o-mini | OpenAI |

### Alias 규칙

자주 쓰는 모델에 짧은 별명을 부여합니다. **Routing → Aliases** 탭에서 설정합니다.

> **[스크린샷]** 대시보드 → Routing → Aliases 탭 화면

| 별칭 | 대상 모델 |
|------|----------|
| fast | gpt-4o-mini |
| smart | claude-sonnet |
| code | claude-sonnet |

### Pattern 규칙

요청 메타데이터나 내용에 따라 모델을 선택합니다. **Routing → Pattern Rules** 탭에서 관리합니다.

> **[스크린샷]** 대시보드 → Routing → Pattern Rules 탭 화면

| 패턴 | 모델 | 우선순위 |
|------|------|---------|
| `code/*` | claude-sonnet | 10 |
| `translate/*` | gpt-4o-mini | 5 |

- `priority`가 높은 규칙이 먼저 평가됩니다
- glob 패턴 문법을 지원합니다

### Auto 라우팅

`model: "auto"`로 요청하면 분류기가 복잡도를 판단합니다. **Routing → Auto** 탭에서 분류기 전략과 복잡도별 모델을 설정합니다.

> **[스크린샷]** 대시보드 → Routing → Auto 설정 화면

| 분류기 전략 | `static` / `llm` / `ml` |
|------------|------------------------|
| **simple** 요청 | gpt-4o-mini |
| **moderate** 요청 | claude-sonnet |
| **complex** 요청 | claude-opus |

## 대시보드에서 관리

웹 대시보드의 **Rules** 페이지에서 라우팅 규칙을 시각적으로 관리할 수 있습니다:

> **[스크린샷]** 대시보드 → Rules 페이지 — Routing Simulator 화면

1. 규칙 추가/수정/삭제
2. 우선순위 드래그 앤 드롭
3. **Routing Simulator**에서 테스트 요청으로 어떤 모델이 선택되는지 확인
