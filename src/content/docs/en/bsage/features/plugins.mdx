## Overview

BSage provides **18 plugins** to integrate with a wide range of external services. Plugins are organized into three categories: Input (collection), Process (transformation), and Output (delivery).

![BSage Dashboard Plugin Manager](/images/screenshots/bsage-dashboard.png)

## Plugin Setup Guide

### Using the Plugin Manager

1. Click **Plugin Manager** in the left menu.
2. Find the plugin you want to configure.
3. Toggle **Enable** on.
4. Fill in the required information in the settings panel.
5. Click **Test Connection** to verify.
6. Click **Save**.

### Credential Setup Details

#### Telegram

| Setting | Value | How to Get It |
|---------|-------|---------------|
| **Bot Token** | `123456789:ABCdef...` | Create a bot via `/newbot` command with [@BotFather](https://t.me/BotFather) on Telegram |

Setup steps:
1. Start a conversation with @BotFather on Telegram.
2. Use the `/newbot` command and set a bot name.
3. Copy the issued Bot Token.
4. Paste it into the Telegram plugin's Bot Token field in the BSage Plugin Manager.
5. After saving, send a message to your bot -- BSage collects it automatically.

#### Slack

| Setting | Value | How to Get It |
|---------|-------|---------------|
| **Bot Token** | `xoxb-...` | Create an app at [Slack API](https://api.slack.com/apps) and issue a Bot Token |
| **Signing Secret** | `abc123...` | Found in the Slack App's Basic Information |
| **Channel** | `#general` | Select the Slack channel to monitor |

Setup steps:
1. Click **Create New App** at [api.slack.com/apps](https://api.slack.com/apps).
2. Select **From scratch** and specify the app name and workspace.
3. Under **OAuth & Permissions**, add the `channels:history` and `channels:read` scopes.
4. Click **Install to Workspace** and authorize.
5. Copy the Bot User OAuth Token and enter it in BSage.
6. Copy the Signing Secret from **Basic Information** and enter it.

#### Discord

| Setting | Value | How to Get It |
|---------|-------|---------------|
| **Bot Token** | `MTk...` | Create a bot at the [Discord Developer Portal](https://discord.com/developers/applications) |

Setup steps:
1. Click **New Application** on the Discord Developer Portal.
2. Under the **Bot** menu, click **Add Bot**.
3. Copy the Token and enter it in BSage.
4. Under **OAuth2 > URL Generator**, select the `bot` scope and `Read Messages` permission.
5. Use the generated URL to invite the bot to your server.

#### Email

| Setting | Value | Example |
|---------|-------|---------|
| **IMAP server** | Your email provider's IMAP address | `imap.gmail.com` |
| **Port** | IMAP port (SSL) | `993` |
| **Email** | Email address to receive from | `user@gmail.com` |
| **Password** | Email password or app password | (app password recommended) |
| **Folder** | Mail folder to monitor | `INBOX` |

:::caution[Gmail Users]
Gmail blocks IMAP access with regular passwords. You must generate an **app password**:
1. Go to Google Account > Security > Enable 2-Step Verification
2. Generate an app password (for Mail)
3. Enter the 16-character password in BSage
:::

## Plugin Pipelines

Combine multiple plugins to build automation pipelines.

### Pipeline Examples

**Example 1: News Monitoring**
```
RSS (collect) → Summarizer (summarize) → Tagger (tag) → Notion (store)
```
New articles from RSS feeds are automatically summarized, tagged, and saved to a Notion database.

**Example 2: Team Communication Archive**
```
Slack (collect) → Deduplicator (deduplicate) → Tagger (tag) → Vault (save to Obsidian)
```
Important Slack conversations are collected, deduplicated, tagged, and stored as Obsidian notes.

**Example 3: Email Intelligence**
```
Email (collect) → DangerAnalyzer (risk analysis) → Summarizer (summarize) → Webhook (alert)
```
Incoming emails are analyzed for risk, summarized, and an alert is sent via Webhook.

## Plugin Status Monitoring

Monitor each plugin's status in real time from the Plugin Manager:

| Status | Meaning | Display |
|--------|---------|---------|
| **Active** | Running normally | Green badge |
| **Error** | Connection error or expired credential | Red badge + error message |
| **Disabled** | Disabled by user | Gray badge |
| **Rate Limited** | API rate limit reached | Yellow badge |

Each plugin card shows the last sync time and number of processed items.

## Plugin Security

BSage keeps plugin credentials safe:

- All credentials are stored **encrypted**.
- Credentials are masked in the dashboard (`sk-...****`).
- Deleting a plugin also deletes its credentials.
- Each plugin follows the principle of least privilege, requesting only the API scopes it needs.
